# 「今天吃什么」App 需求文档

## 一、产品背景

开发者自用 + 家人使用，解决每天/每周不知道吃什么的问题。  
长期目标：若有需求，可对外开放，实现商业化。

---

## 二、核心问题

- 每天或每周末不知道吃什么
- 没有一个地方集中管理自己常做的菜、常点的外卖
- 缺乏提前规划三餐的工具

---

## 三、目标用户

- **第一阶段**：开发者本人 + 家人（私用）
- **第二阶段**：有相同困扰的普通用户（对外开放）

---

## 四、核心功能（V1）

### 4.1 最核心功能（MVP）

| 功能 | 描述 |
|------|------|
| 上传菜谱 | 上传照片 + 填写做法步骤，记录自己做过的菜或网上学的菜 |
| 外卖记录 | 记录常点的外卖，区别于自己做的菜 |
| 选择三餐 | 从菜谱库中选择明天/本周的早中晚餐 |
| 今日展示 | 首页展示今天三餐计划，一目了然 |

### 4.2 菜谱模块

- 菜谱分类：**自己做** / **外卖**
- 每道菜包含：
  - 菜名
  - 封面照片（自己拍的）
  - 详细做法步骤
  - 标签（如：早餐、快手菜、家常菜等）
  - 备注
- 支持浏览、搜索、筛选

### 4.3 餐计划模块

- 按日期规划三餐（早 / 中 / 晚）
- 从菜谱库中选择对应的菜
- 已选中的菜在首页展示
- 支持查看历史计划

### 4.4 首页

- 展示**今天**的三餐计划
- 快速跳转到菜谱详情
- 如未规划，引导用户去选菜

---

## 五、页面结构

```
/              首页        → 今天吃什么（展示三餐计划）
/recipes       菜谱库      → 所有菜的列表/网格展示
/recipes/new   上传菜谱    → 填写菜名、步骤、上传图片
/recipes/:id   菜谱详情    → 图片 + 完整做法
/plan          餐计划      → 选择日期，为三餐选菜
```

---

## 六、数据结构

### recipes（菜谱表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 主键 |
| name | string | 菜名 |
| type | enum | `homemade`（自己做）/ `takeout`（外卖） |
| image_url | string | 封面图片地址 |
| description | string | 简介 |
| steps | json | 做法步骤数组 |
| tags | string[] | 标签列表 |
| created_at | timestamp | 创建时间 |

### meal_plans（餐计划表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 主键 |
| date | date | 计划日期 |
| meal_type | enum | `breakfast` / `lunch` / `dinner` |
| recipe_id | uuid | 关联菜谱 |
| user_id | uuid | 关联用户 |

## 七、技术选型

| 层级 | 技术 | 理由 |
|------|------|------|
| 前端 | React | 开发者熟悉，生态丰富 |
| 后端/数据库 | Supabase | 无需写后端，自带数据库+存储+Auth |
| 图片存储 | Supabase Storage | 与数据库一体，简单直接 |
| 部署 | Vercel | 免费，一键部署，与 React 配合最佳 |

---

## 八、开发阶段规划

### Phase 1 — MVP（当前目标）

- [ ] UI 原型设计（React 静态页面）
- [ ] 搭建 React 项目 + 路由
- [ ] 接入 Supabase，建表
- [ ] 上传菜谱功能
- [ ] 菜谱列表 + 详情页
- [ ] 餐计划选择
- [ ] 首页「今天吃什么」展示

### Phase 2 — 多用户（商业化准备）

- [ ] 用户注册/登录（Supabase Auth）
- [ ] 数据隔离（每个用户只看自己的菜谱）
- [ ] 移动端适配优化
- [ ] 分享功能（把菜谱分享给朋友）

### Phase 3 — 商业化探索

- [ ] 公开菜谱广场（用户可以发现别人的菜谱）
- [ ] 菜谱收藏 / 点赞
- [ ] 智能推荐（根据历史选择推荐今天吃什么）
- [ ] 考虑自建后端（如 Node.js）替换 Supabase

---

## 九、非功能需求

- **移动端优先**：主要在手机上使用
- **速度**：图片上传体验流畅
- **简单**：界面不复杂，家人也能用

---

## 十、暂不做（V1 排除）

- 营养计算
- 购物清单
- 社区/评论功能
- AI 推荐
- 多语言
